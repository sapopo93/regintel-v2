RegIntel v2 - App Summary

What It Is

RegIntel v2 helps care providers prove inspection readiness with evidence before inspectors arrive, without guesswork or checklist theater.

Who It Is For

Primary: UK CQC-registered care providers (small to mid-sized) preparing for first inspection or re-inspection.

What It Does

  • Regulation intelligence with versioned CQC regs and change detection.
  • Policy mapping from provider policies to specific regulation clauses.
  • Provider context snapshots that freeze the inspection context in time.
  • Mock inspection engine with bounded, topic-driven questioning and scoring.
  • Evidence management with secure uploads and proof-of-existence.
  • Remediation actions tied to findings and verified closure evidence.
  • Inspection confidence reports (likely pass/fail and why).

How It Works (Repo Evidence)

  • UI: Next.js App Router in apps/web is a projection layer and calls the backend via a type-safe API client.
  • API: Express REST server in apps/api exposes providers, topics, mock sessions, findings, evidence, audit, and export endpoints.
  • Domain logic: shared modules in packages/domain (e.g., mock inspection engine, policy intelligence, readiness export).
  • Auth: Clerk SignIn UI + API JWT verification + Clerk webhooks.
  • Evidence storage: filesystem blob store at /var/regintel/evidence-blobs with SHA-256 addressing and quarantine.
  • Data store: in-memory store for most tests; PostgreSQL with RLS used for tenant isolation tests and migrations exist.

How To Run (Minimal)

  • Install: pnpm install
  • Configure: copy .env.example to .env and set Clerk keys (and CQC API key if needed).
  • Start API: pnpm api:dev (defaults to http://localhost:3001)
  • Start UI: pnpm web:dev (open http://localhost:3000)
